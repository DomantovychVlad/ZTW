# server/ — самостійно-хостований бекенд

Три логічні частини, що піднімаються одним скриптом/контейнером. Деталі —
[../docs/architecture.md](../docs/architecture.md), розгортання — PRD 4 і 5.11.

| Частина | Призначення |
|---------|-------------|
| **rendezvous** | Реєстр онлайн-пристроїв, зведення за ID, сигналінг WebRTC (SDP/ICE). Вмісту сесії не бачить. |
| **relay** | `coturn` (TURN, BSD) — запасний канал, коли P2P неможливий. Бачить лише зашифровані байти. |
| **accounts** | Акаунти (e-mail+пароль, 2FA), адресна книга, синхронізація між клієнтами. |

- Основне розгортання: **Oracle Cloud Always Free, ARM Ampere A1** (не micro).
- Також: орендований VPS із доменом; домашнє залізо з динамічним DNS.
- Автогенерація ключів сервера на першому запуску; масштаб — додаткові relay в інших регіонах.

**Статус:** проєктування (Етап 1). Технологію бекенду фіксуємо на старті Етапу 1.
