# Ліцензійна політика та чистота коду

ZortilWatch — **закритий комерційний продукт**, який має лишатися вільним для
продажу й поширення у закритому вигляді. Цей документ фіксує правила залежностей і
автоматичний фільтр (рішення **F1**).

## 1. Принцип «вивчати, не копіювати»

Власний код — це наша власна реалізація. Код під копілефт-ліцензіями (**GPL, AGPL,
LGPL**), зокрема **RustDesk та його сервер**, **не копіюється, не переноситься й не
переписується рядок у рядок**. Дозволено лише вивчення публічно документованих
архітектурних рішень і загальних технічних принципів, що не є об'єктом авторського
права.

## 2. Політика F1 — що дозволено

**Дозволені** (пермісивні + слабкий файловий копілефт, безпечні для закритого продукту):
`MIT`, `Apache-2.0`, `Apache-2.0 WITH LLVM-exception`, `BSD-2-Clause`, `BSD-3-Clause`,
`ISC`, `0BSD`, `Zlib`, `BSL-1.0`, `Unicode-DFS-2016`, `Unicode-3.0`, `CC0-1.0`, **`MPL-2.0`**.
Подвійні вирази `A OR B` дозволені, якщо дозволений хоч один операнд.

**Тверда заборона** (сильний / мережевий копілефт): `GPL-2.0`, `GPL-3.0`, `AGPL-3.0`,
`LGPL-2.1`, `LGPL-3.0` (LGPL — бо Rust лінкує статично, що втягує весь бінарник),
`SSPL-1.0` та подібні.

**Невідоме / відсутні метадані ліцензії** → збірка **падає** й вимагає ручного
розгляду (задокументований список винятків). Ніколи не дозволяється «наосліп».

## 3. Як це enforced (автоматичний фільтр)

- **Rust:** `cargo deny check licenses` із allow-списком вище (модель «усе, що не
  дозволено явно — заборонено»; крейт без метаданих ліцензії падає автоматично).
  Конфіг — у `deny.toml` у корені (додається разом із Cargo-воркспейсом).
- **npm/TS:** `license-checker-rseidelsohn` (сам під BSD-3) з `--onlyAllow "<той самий
  список>"`; падає на першій недозволеній або невідомій ліцензії.
- Обидва запускаються як перед-білд-крок (без хмарного CI).

### Відомі уточнення (clarify / exceptions)
- `ring` має порожні метадані та вираз `MIT AND ISC AND OpenSSL` → потрібен `clarify`
  з фіксованим хешем LICENSE і **виняток на `OpenSSL` лише для `ring`** (OpenSSL-ліцензія
  безпечна для закритого продукту; не дозволяємо її глобально).
- `webpki` (`ISC`) → `clarify` із фіксованим хешем.

## 4. Що автоматичні сканери НЕ ловлять (важливо для продажу)

1. **Патентні роялті H.264.** Ліцензія коду `openh264` — BSD (чисто), але **патенти
   H.264 (MPEG-LA/Via-LA)** — окреме комерційне зобов'язання. Роялті покриті лише
   при постачанні **офіційного бінарника Cisco**; самостійна збірка з джерел переносить
   відповідальність на нас. Чистий від роялті шлях — **VP9/AV1** (`rav1e`, BSD).
2. **MPL-2.0** — якщо ми **модифікуємо** MPL-файл, треба надати джерело саме цього
   файла (не всього продукту). Процесне зобов'язання, якого сканер не контролює.
3. **NVIDIA/AMD/Intel SDK** (NVENC/AMF) — Rust-біндинги MIT, але самі SDK пропрієтарні.
   Тому базовий шлях кодування — Media Foundation, без цих SDK.

> Перед першим комерційним продажем варто, щоб **юрист** перевірив пункти 1–3 і
> загальну політику (особливо MPL і патенти H.264).
