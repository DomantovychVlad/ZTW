# Пакування Windows (Етап 11)

ZortilWatch постачається у двох рівноцінних форматах: **інсталятор** (NSIS) і
**портативна** версія (тека з .exe). Обидва запускають той самий бінарник із вшитим
фронтендом — функціональність ідентична; різниця лише в способі запуску й зберіганні
налаштувань.

## Передумови

- Rust toolchain (як у `rust-toolchain.toml`), Node 20+.
- **Microsoft Edge WebView2 Runtime** на цільовій машині. Є за замовчуванням на
  Windows 10 1803+/11. Інсталятор за потреби докачує його (`downloadBootstrapper`);
  для портативної версії — передумова (один раз на машину).
- Для NSIS-збірки Tauri CLI сам докачує NSIS під час першого `tauri build`.

## Інсталятор (NSIS)

```
npm run bundle:win
```

Артефакт: `target/release/bundle/nsis/ZortilWatch_<версія>_x64-setup.exe`.

Налаштування у `desktop/src-tauri/tauri.conf.json` → `bundle.windows.nsis`:

- `installMode: currentUser` — встановлення **без прав адміністратора** (per-user), як
  і автозапуск (`HKCU\…\Run`). Ярлики в меню «Пуск» і на робочому столі, коректне
  оновлення поверх попередньої версії, запис до «Програми та засоби».
- `languages: [English, Ukrainian]` з вибором мови майстра.
- Фонова присутність — не окрема служба, а автозапуск при вході (тумблер у застосунку,
  Рівень A). Служба до входу/екрана входу (Рівень B) — поза цим етапом.

## Портативна версія

```
npm run bundle:portable
```

Артефакт: тека `dist-portable/ZortilWatch/` (готова до копіювання на флешку):

- `ZortilWatch.exe` — запуск без встановлення.
- `ZortilWatch.portable` — маркер; його наявність вмикає портативний режим: дані
  WebView2 (адреса сервера, ідентичність пристрою, постійний пароль) пишуться в
  `.\data` поряд із .exe, **а не в `%LOCALAPPDATA%`**. Видалили теку — слідів немає.
- `zortilwatch.config.json.sample` — шаблон для націлення на власний сервер (нижче).
- `README.txt` — коротка інструкція для кінцевого користувача.

Обмеження: справжній **єдиний** .exe неможливий через залежність від WebView2 Runtime
(рішення зафіксовано в [architecture.md](architecture.md) §4) — портатив це тека, а не
один файл.

## Налаштування клієнта на власний сервер (PRD 5.11)

Адреса сервера задається у вікні входу й зберігається локально. Щоб **постачати** клієнт
уже націленим на свій бекенд, покладіть поряд із .exe файл `zortilwatch.config.json`:

```json
{ "server": "https://your-server.example:8787" }
```

Він задає типову адресу для першого запуску (далі користувач може змінити її у вікні
входу). Окремого «публічного ключа сервера» немає **навмисно**: модель довіри B1 —
наскрізне шифрування сесії через пароль-PAKE, тож rendezvous-сервер ненадійний за
побудовою й не може ні прочитати, ні підмінити сесію. Пінити нема чого; єдине
налаштування — адреса. (Детально — [architecture.md](architecture.md), рішення B1.)

## Підпис коду

Збірки **не підписані**. Для продажу потрібен сертифікат OV/EV — інакше SmartScreen
лякатиме користувачів. Підпис налаштовується в `bundle.windows` (`certificateThumbprint`,
`signCommand`) і потребує сертифіката від користувача — поза автоматичною збіркою.
