# core/ — Rust-ядро

Продуктивне ядро, спільне для всіх платформ: захоплення екрана, апаратне
кодування/декодування, передавання вводу, мережа (WebRTC + обхід NAT) та
наскрізне шифрування. Деталі стека — [../docs/architecture.md](../docs/architecture.md).

Модулі `core/src` (крейт `zortilwatch-core`).

Чиста поверхня (wasm-safe; реюзає веб-клієнт через WASM, без мережі/ОС-залежностей):

| Модуль | Призначення |
|--------|-------------|
| `crypto` | Наскрізне крипто сесії — PAKE на паролі + прив'язка DTLS-відбитків (B1) |
| `session` | Встановлення захищеної сесії (B1, PRD 5.1/5.10) |
| `media` | Фреймінг H.264 access units для datachannel |

Нативна поверхня (за фічею `native`, default; поза wasm — мережа/ОС):

| Модуль | Призначення |
|--------|-------------|
| `net` | Мережеве ядро на str0m (WebRTC) |
| `signal` | Клієнт сигналінгу (WebSocket до `/signal`) |
| `relay` | Одно-сокетний TURN/ICE (ретранслятор) |
| `connect` | Стратегія «спершу напряму, ретранслятор як запас» (E1) |
| `connection` | Оркестрація сесії — API для оболонки (Tauri/UI) і нативного керованого |
| `capture` | Захоплення екрана (WGC; DXGI-DDA для Tier B) |
| `encode` | Кодування екрана в H.264 через Media Foundation |
| `input` | Модель подій вводу (пульт -> керований) |
| `clipboard` | Спільний буфер обміну (PRD 5.8) |
| `files` | Передача файлів (PRD 5.7) |
| `blank` | Затемнення екрана керованого під час сесії (PRD 5.10) |
| `wol` | Wake-on-LAN (PRD 5.9) — побудова й надсилання «магічного пакета» |
| `identity` | Ідентичність пристрою (публічний ID + client_secret + псевдонім) |
| `password` | Паролі підключення (PRD 5.1) |

> Декодування на боці пульта робить браузер/WebView2 WebRTC — у ядрі його немає.

Windows-служба (Tier B: керований на екрані входу / secure-desktop) — це окремий крейт
[`../service`](../service) (`zortilwatch-service`), а не модуль ядра.
