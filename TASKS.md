# ZortilWatch — Етапи робіт і завдання

Цей файл — декомпозиція [PRD.md](PRD.md) на етапи робіт і конкретні завдання.
Порядок етапів відбиває інженерні залежності (що від чого залежить), а не порядок
розділів у PRD. Обсяг MVP та подальших версій взято з розділу 7 PRD.

## Як читати цей файл

- `- [ ]` — завдання не виконане, `- [x]` — виконане.
- Кожен етап має **Мету**, **Залежності**, список **Завдань** і критерій **Готово коли**.
- У дужках після завдань вказано розділ PRD-джерела, напр. `(PRD 5.1)`.
- Етапи 0–11 складають **MVP** (перша версія). Етап 12 — **подальші версії** (post-MVP).

| Етап | Назва | Блок | Залежить від |
|------|-------|------|--------------|

## Етап 0 — Фундамент проєкту та ліцензійна гігієна

**Мета:** підготувати структуру репозиторію, інструментарій і правила, що
гарантують юридичну чистоту продукту (лише пермісивні ліцензії). (PRD 2, 6)

**Залежності:** немає.

### Завдання
- [x] Створити структуру монорепозиторію: `core/` (Rust), `ui/` (TS+React), `desktop/` (Tauri), `server/` (бекенд), `docs/`. (PRD 6)
- [x] Зафіксувати версії стека: Rust toolchain, Node/TS, React, Tauri, WebRTC-стек. (PRD 6)
- [x] Налаштувати форматери, лінтери й білд-скрипти для Rust і TS (єдиний `make`/`just`/npm-скрипт). (PRD 6)
- [x] Завести процес перевірки ліцензій залежностей: дозволені пермісивні + MPL-2.0 (політика F1, ширша за початкову MIT/Apache/BSD — див. `docs/licensing.md`); автоматична відмова на GPL/AGPL/LGPL (`cargo-deny`, перевірка npm-ліцензій). (PRD 2)
- [x] Зафіксувати у `docs/` правило: код під GPL/AGPL (зокрема RustDesk і його сервер) не копіюється й не переписується; дозволено лише вивчення архітектурних принципів. (PRD 2)
- [x] Описати архітектуру «спільне ядро + спільний UI» в `docs/architecture.md`. (PRD 5.14, 6)
- [x] Налаштувати локальне середовище розробки без зовнішніх хмарних залежностей.

**Готово коли:** репозиторій збирається порожнім каркасом на всіх трьох поверхнях
(core, ui, desktop); перевірка ліцензій проходить і блокує копілефт; архітектурний
документ затверджено.

## Етап 12 — Подальші версії (post-MVP)

**Мета:** розширити продукт на інші платформи та додати функції сесії. (PRD 3, 5.14, 7)

**Залежності:** усі MVP-етапи.

### Платформи
- [ ] **macOS** — обидві ролі (пульт і керований). (PRD 3, 5.14)
- [ ] **Linux** — обидві ролі. (PRD 3, 5.14)
- [ ] **Android** — повноцінно як пульт; як керований із застереженнями (дозвіл на захоплення екрана, служба спеціальних можливостей / механізми виробника). (PRD 3, 5.14)
- [ ] **iOS / iPadOS** — лише роль пульта (Apple не дозволяє віддалений системний ввід). (PRD 3, 5.14)
- [ ] Спільний акаунт, спільна адресна книга й однаковий принцип з'єднання на всіх платформах. (PRD 5.14)

### Додаткові функції сесії
- [ ] **Чат** під час сесії. (PRD 7)
- [ ] **Передавання звуку** з віддаленого пристрою. (PRD 7)
- [ ] **Запис сесій**. (PRD 7)

**Готово коли:** кожна нова платформа працює у відведених їй ролях зі спільним

## Наскрізні принципи (діють на всіх етапах)

- **Ліцензійна чистота:** лише пермісивні залежності; жодного копіювання GPL/AGPL-коду. (PRD 2)
- **Безпека за замовчуванням:** шифрування й контроль доступу не є опціями. (PRD 5.10)
- **Незалежність від третіх сторін:** уся інфраструктура належить власнику й переноситься між провайдерами. (PRD 1, 5.11)
- **Спільне ядро + спільний UI:** максимальне повторне використання коду між платформами. (PRD 5.14, 6)
- **Простота розгортання сервера:** скрипт/контейнер, автогенерація ключів, мінімум ручних дій. (PRD 4, 5.11)
